# Beyond Homophily: Benchmarking and Enhancing Structural Explainability in Heterophilic Graph Neural Networks

This repository is designed to benchmark existing GNN explainers (like GNNExplainer and PGExplainer) on heterophilic graphs and introduce a modified objective function to explicitly account for "disassortative mixing" or incorporate node feature importance more heavily.

## Project Structure

```text
hetero-gnn-explainability/
├── data/                      # Data output (Ignored by git)
│   ├── raw/                   # Generated by ShapeGGen
│   └── processed/             # Pre-computed features/masks
├── notebooks/                 # Jupyter notebooks for visualization
│   ├── 01_data_exploration.ipynb
│   └── 02_visualize_explanations.ipynb
├── src/                       # Core Python modules
│   ├── data_gen/              # Data generation wrapping GraphXAI ShapeGGen
│   ├── models/                # GNN Model definitions (GCN, GraphSAGE)
│   ├── explainers/            # Explainers: Baselines and Novel HeteroGNNExplainer
│   └── utils/                 # Metrics and plotting tools
├── scripts/                   # Executable scripts for the pipeline
│   ├── 01_generate_datasets.py
│   ├── 02_train_models.py
│   └── 03_run_benchmarks.py
├── results/                   # Saved model weights, plots, and metrics
└── requirements.txt           # Environment dependencies
```

## Implementation Phases

1. **Environment Setup and Data Generation**
   - Utilizing GraphXAI's `ShapeGGen` to generate datasets with tunable homophily.
   - Script: `scripts/01_generate_datasets.py`
2. **Train the "Black Box" Models**
   - Training homophily-biased (GCN) and heterophily-aware (GraphSAGE) models on generated data.
   - Script: `scripts/02_train_models.py`
3. **Baseline Benchmarking**
   - Running GNNExplainer and PGExplainer on trained models.
   - Script: `scripts/03_run_benchmarks.py`
4. **The Novel Contribution**
   - Implementing a Heterophily-Aware Explainer (`src/explainers/hetero_explainer.py`).
5. **Visualization and Reporting**
   - Using Jupyter notebooks to visualize the explained subgraphs vs baseline explainers.
